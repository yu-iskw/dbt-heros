# Documentation Basics

dbt has a built-in documentation system for describing every resource.

## Doc Blocks

Reusable markdown descriptions in `.md` files:

```markdown
{# docs/column_descriptions.md #}

{% docs order_id %}
Unique identifier for each order, generated by the source system.
This is the primary key and should never be null.
{% enddocs %}

{% docs customer_email %}
The customer's email address. May be null for guest checkouts.
{% enddocs %}
```

Referencing in YAML:

```yaml
models:
  - name: fct_orders
    columns:
      - name: order_id
        description: '{{ doc("order_id") }}'
```

## Exposures

Define downstream consumers of dbt models:

```yaml
# models/exposures.yml
version: 2

exposures:
  - name: weekly_sales_dashboard
    label: Weekly Sales Dashboard
    type: dashboard
    maturity: high
    url: https://bi.tool/dashboards/42
    description: >
      Weekly sales performance dashboard used by the sales team.
    depends_on:
      - ref('fct_orders')
      - ref('dim_customers')
    owner:
      name: Jane Analyst
      email: jane@company.com
```

### Exposure Types

| Type | Description |
|:--|:--|
| `dashboard` | BI dashboards |
| `notebook` | Jupyter / data science notebooks |
| `analysis` | Ad-hoc analysis queries |
| `ml` | Machine learning models |
| `application` | Production applications |

### Required Properties

| Property | Description |
|:--|:--|
| `name` | Unique exposure identifier |
| `type` | One of the types above |
| `depends_on` | List of `ref()`, `source()`, or `metric()` |
| `owner` | Must have `name` and/or `email` |

## Generating Documentation

```bash
dbt docs generate    # creates target/catalog.json and target/manifest.json
dbt docs serve       # launches local documentation website
```

## Persist Docs

Push descriptions into the warehouse catalog:

```yaml
models:
  - name: fct_orders
    config:
      persist_docs:
        relation: true
        columns: true
```

## Selection by Exposure

```bash
dbt run -s +exposure:weekly_sales_dashboard   # run all upstream dependencies
dbt test -s +exposure:weekly_sales_dashboard  # test all upstream dependencies
```
